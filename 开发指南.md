# Resume Printer 项目开发指南

一个以"复古机械打字机"为视觉与交互核心的个人简历展示系统，支持Web端和微信小程序双端展示，具有独特的沉浸式体验。

---

## 1. 项目概述

### 1.1 项目简介
Resume Printer 是一个多端简历展示系统，包含 Web 版本和微信小程序版本，采用数据驱动的内容管理方式，支持通过 GitHub 仓库直接维护简历内容。

### 1.2 核心特性
- **复古打字机视觉效果**：米黄色纸张背景、打字机滚筒装饰、等宽字体、闪烁光标
- **数据驱动内容**：从 `resume-data.json` 异步获取数据并动态渲染
- **打字机交互效果**：主标题自动逐字打印、点击导航后内容切换的打字/清除效果
- **GitHub 内容管理**：通过修改仓库 JSON 文件更新简历内容
- **音效系统**：打字音效及开关控制
- **PDF 导出功能**：将简历内容导出为 PDF 文档
- **响应式设计**：适配桌面和移动设备
- **留言板功能**：支持访客留言（后端 API）
- **双端支持**：Web 端和微信小程序端

## 2. 技术栈选择

### 2.1 Web 端技术栈
- **前端框架**：原生 HTML5 + CSS3 + JavaScript ES6+
  - 理由：轻量级、无构建依赖、易于维护和部署
- **样式方案**：原生 CSS3
  - 理由：无需预处理器，直接控制视觉效果
- **字体库**：Google Fonts（Special Elite + Courier Prime）
  - 理由：提供复古打字机字体效果
- **PDF 导出**：jsPDF + html2canvas
  - 理由：纯前端 PDF 生成，无需后端支持
- **部署平台**：GitHub Pages
  - 理由：免费、自动化部署、支持自定义域名

### 2.2 微信小程序技术栈
- **开发框架**：微信小程序原生开发
  - 理由：官方支持、性能最优、API 完整
- **样式方案**：WXSS
  - 理由：小程序原生样式方案
- **开发工具**：微信开发者工具
  - 理由：官方开发环境，支持调试和预览

### 2.3 后端技术栈
- **Java 后端**：Java 8 + Spring Boot
  - 理由：稳定可靠、企业级应用标准
- **部署平台**：阿里云函数计算
  - 理由：按需付费、自动扩缩容、无需服务器管理
- **本地服务器**：Node.js HTTP Server
  - 理由：快速本地开发和测试

## 3. 本地运行步骤

### 3.1 Web 端本地运行

#### 方式一：使用 Node.js 服务器（推荐）

1. **克隆仓库**
```bash
git clone https://github.com/[你的用户名]/resume-printer.git
cd resume-printer
```

2. **启动服务器**
```bash
# Windows
node server.js

# 或使用 PowerShell
.\start-server.ps1
```

3. **访问网站**
打开浏览器访问：`http://localhost:3000`

#### 方式二：直接打开 HTML 文件

1. **克隆仓库**
```bash
git clone https://github.com/[你的用户名]/resume-printer.git
cd resume-printer
```

2. **打开文件**
直接在浏览器中打开 `index.html` 文件

**注意**：直接打开 HTML 文件时，某些功能（如从 GitHub 加载数据）可能受到浏览器 CORS 限制。

### 3.2 微信小程序本地运行

1. **安装微信开发者工具**
   - 下载地址：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html

2. **克隆仓库**
```bash
git clone https://github.com/[你的用户名]/resume-printer.git
cd resume-printer/wechat-miniprogram
```

3. **配置 AppID**
   - 打开 `project.config.json` 文件
   - 将 `"appid": "请填写您的AppID"` 替换为您的小程序 AppID

4. **打开项目**
   - 启动微信开发者工具
   - 选择"导入项目"
   - 选择 `wechat-miniprogram` 目录
   - 填写 AppID
   - 点击"导入"

5. **编译运行**
   - 点击"编译"按钮
   - 在模拟器中查看效果

### 3.3 后端服务本地运行

#### Java 后端（阿里云函数计算本地测试）

1. **准备环境**
   - 安装 Java 8 或更高版本
   - 确保已安装 Maven 或 Gradle（如需重新编译）

2. **启动服务**
```bash
cd backend/deploy

# Windows
start.bat

# Linux/Mac
bash start.sh
```

3. **测试 API**
   - 访问：`http://localhost:9000/messages`
   - 测试留言功能

## 5. 核心功能模块说明

### 5.1 Web 端核心模块

#### 5.1.1 数据获取模块（dataFetcher.js）
- **功能**：从 GitHub 或本地加载简历数据
- **主要方法**：
  - `fetchResumeData()`: 获取简历数据
  - `loadLocalData()`: 加载本地数据
- **配置项**：数据源 URL、缓存策略

#### 5.1.2 打字机效果模块（typewriter.js）
- **功能**：实现打字机视觉效果
- **主要方法**：
  - `typeText(element, text, speed)`: 逐字打印文本
  - `deleteText(element, speed)`: 逐字删除文本
  - `setCursorBlink(enabled)`: 控制光标闪烁
- **配置项**：打字速度、删除速度、光标样式

#### 5.1.3 内容渲染模块（contentRenderer.js）
- **功能**：根据数据动态渲染页面内容
- **主要方法**：
  - `renderSection(sectionName, data)`: 渲染指定区块
  - `renderExperience(data)`: 渲染工作经历
  - `renderSkills(data)`: 渲染技能列表
- **配置项**：渲染模板、动画效果

#### 5.1.4 音效管理模块（soundManager.js）
- **功能**：管理打字音效
- **主要方法**：
  - `playTypeSound()`: 播放打字音效
  - `toggleSound()`: 切换音效开关
  - `setVolume(level)`: 设置音量
- **配置项**：音效文件路径、默认音量

#### 5.1.5 PDF 导出模块（pdfExporter.js）
- **功能**：将简历导出为 PDF 文档
- **主要方法**：
  - `exportToPDF()`: 导出 PDF
  - `generatePDF(content)`: 生成 PDF 内容
- **依赖库**：jsPDF、html2canvas

#### 5.1.6 留言管理模块（messageManager.js）
- **功能**：管理访客留言
- **主要方法**：
  - `fetchMessages()`: 获取留言列表
  - `submitMessage(message)`: 提交新留言
  - `displayMessages(messages)`: 显示留言
- **API 端点**：`/messages`

### 5.2 微信小程序核心模块

#### 5.2.1 页面模块
- **index**: 首页，展示个人信息概览
- **about**: 关于我，详细个人介绍
- **experience**: 工作经历，支持展开/收起
- **education**: 教育背景，学历信息
- **skills**: 技能专长，进度条展示
- **projects**: 项目经验，项目案例
- **contact**: 联系方式，一键拨号和发邮件

#### 5.2.2 工具模块（utils/）
- **data.js**: 数据处理工具
- **eventBus.js**: 事件总线
- **typewriter.js**: 打字机效果
- **util.js**: 通用工具函数

const DATA_SOURCE_URL = 'https://raw.githubusercontent.com/[你的用户名]/[你的仓库]/main/resume-data.json';

const TYPEWRITER_CONFIG = {

## 8. 开发规范

### 8.1 代码风格

#### JavaScript
- 使用 ES6+ 语法
- 使用 const/let 替代 var
- 使用箭头函数
- 使用模板字符串
- 添加必要的注释

#### CSS
- 使用 BEM 命名规范
- 使用 CSS 变量
- 响应式设计优先
- 避免使用 !important

#### 小程序
- 遵循小程序官方规范
- 使用小程序组件 API
- 合理使用生命周期函数

### 8.2 Git 提交规范

提交信息格式：
```
<type>(<scope>): <subject>

<body>

<footer>
```

类型（type）：
- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具相关

示例：
```
feat(typewriter): 添加打字机音效开关功能

- 添加音效开关按钮
- 实现音效状态持久化
- 更新用户界面

Closes #123
```

### 8.3 文件命名规范

- HTML 文件：小写字母，使用连字符（kebab-case）
- CSS 文件：小写字母，使用连字符（kebab-case）
- JavaScript 文件：小写字母，使用连字符（kebab-case）
- 小程序页面：小写字母，使用连字符（kebab-case）

## 9. 常见问题

### 9.1 Web 端问题

**Q: 打字机效果不显示？**
A: 检查浏览器控制台是否有错误，确保 JavaScript 文件正确加载。

**Q: 数据加载失败？**
A: 检查数据源 URL 是否正确，确保 GitHub 仓库是公开的。

**Q: 音效无法播放？**
A: 浏览器可能阻止自动播放，需要用户交互后才能播放音效。

**Q: PDF 导出失败？**
A: 确保已引入 jsPDF 和 html2canvas 库，检查浏览器兼容性。

### 9.2 微信小程序问题

**Q: 编译失败？**
A: 检查 AppID 是否正确，确保所有页面路径在 app.json 中配置。

**Q: 样式不显示？**
A: 检查 WXSS 文件路径，确保样式文件正确引用。

**Q: 真机调试失败？**
A: 确保手机和电脑在同一网络，检查开发者工具设置。

### 9.3 后端问题

**Q: 函数计算启动失败？**
A: 检查 JAR 文件是否完整，确保启动命令正确。

**Q: API 返回 500 错误？**
A: 检查函数日志，确认内存配置是否足够。

**Q: 留言数据丢失？**
A: 函数实例重启后内存数据会重置，这是正常行为。

## 10. 性能优化

### 10.1 Web 端优化

1. **资源优化**
   - 压缩 CSS 和 JavaScript 文件
   - 使用图片懒加载
   - 启用浏览器缓存

2. **加载优化**
   - 异步加载非关键资源
   - 使用 CDN 加速
   - 预加载关键资源

3. **渲染优化**
   - 减少 DOM 操作
   - 使用 CSS 动画替代 JavaScript 动画
   - 避免强制同步布局

### 10.2 微信小程序优化

1. **分包加载**
   - 将非核心页面放入分包
   - 减少主包体积

2. **数据优化**
   - 使用本地缓存
   - 减少网络请求
   - 合理使用 setData

3. **渲染优化**
   - 避免频繁 setData
   - 使用虚拟列表处理长列表
   - 优化 WXML 结构
